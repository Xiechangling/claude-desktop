## 项目简介

**Claude Desktop App** 是一个桌面应用程序，让你可以使用任何 AI 模型来运行 Claude Code 的智能代理系统，同时拥有类似 claude.ai 的用户界面。

### 核心特点

- **多模型支持**：兼容任何 OpenAI 格式的 API（Qwen、DeepSeek、Claude、GPT、Gemini 等）
- **完整的代理能力**：19+ 工具，包括文件操作、命令执行、网络搜索等
- **优雅的界面**：仿 claude.ai 的深色模式界面，流式响应，实时工具卡片
- **技能系统**：可复用的指令集，支持自定义技能
- **项目管理**：上传参考文档，跨对话共享知识库

### 适用场景

- 代码开发与调试
- 项目脚手架生成
- 文档编写
- 数据分析
- 自动化任务执行
- 学习编程

## 核心功能

### 1. 多模型智能代理

**功能描述**：连接任何 OpenAI 兼容的 API 提供商，内置代理自动转换 Anthropic 和 OpenAI 格式。

**支持特性**：
- 工具调用（函数调用）
- 扩展思考/推理模式（Qwen `enable_thinking`、DeepSeek reasoning）
- 视觉/图像理解
- 实时流式响应

**支持的提供商**：
| 提供商 | Base URL | 格式 |
|--------|----------|------|
| Qwen (阿里云) | `https://dashscope.aliyuncs.com/compatible-mode` | OpenAI |
| DeepSeek | `https://api.deepseek.com` | OpenAI |
| SiliconFlow | `https://api.siliconflow.cn` | OpenAI |
| OpenAI | `https://api.openai.com` | OpenAI |
| Anthropic | `https://api.anthropic.com` | Anthropic |
| 自定义 | 你的 URL | OpenAI |

### 2. 代理工具能力（19+ 工具）

Claude Code 引擎提供的工具：

#### 文件操作
- **Read**：读取文件内容
- **Write**：创建新文件
- **Edit**：修改现有文件
- **Glob**：按模式搜索文件
- **Grep**：搜索文件内容

#### 系统操作
- **Bash**：执行 Shell 命令
- **Agent**：生成子代理执行并行任务

#### 网络功能
- **WebSearch**：搜索网络
- **WebFetch**：获取网页内容

#### 其他工具
- **Skills**：调用可复用的指令集
- **TaskCreate/TaskUpdate**：任务管理
- **AskUserQuestion**：交互式提问

### 3. 技能系统（Skills）

**功能描述**：可复用的指令集，类似于预设的专家模式。

**内置技能**：
- `skill-creator`：创建自定义技能
- `code-review`：代码审查
- `create-project`：项目脚手架生成
- `doc-writer`：文档生成
- `frontend-design`：前端设计

**使用方式**：
1. 在输入框菜单中浏览和选择技能
2. 输入 `/skill-name` 调用技能（显示为蓝色标签）
3. AI 通过 Skill 工具读取技能指令

**自定义技能**：
- 技能保存在 `~/.claude/skills/` 目录
- 每个技能是一个文件夹，包含 `SKILL.md` 文件
- 使用 `/skill-creator` 创建新技能

**技能目录结构**：
```
~/.claude/skills/
├── skill-creator/
│   └── SKILL.md
├── code-review/
│   └── SKILL.md
└── your-custom-skill/
    └── SKILL.md
```

### 4. 项目管理（Projects）

**功能描述**：上传参考文档，创建跨对话的共享知识库。

**特性**：
- 上传代码、文档、PDF 等文件
- 文件按需通过 Read 工具访问（不占用上下文）
- 项目级指令注入到每个对话
- 支持多个项目切换

**使用场景**：
- 为特定项目提供背景知识
- 共享代码库文档
- 保存常用配置和规范

### 5. 流式用户体验

**实时反馈**：
- 工具操作实时显示（Read / Write / Edit）
- 中间文本与工具卡片按时间顺序交错显示
- 工作完成后自动折叠，只显示最终响应
- 扩展思考模式，可折叠的思考块

**界面特性**：
- 1:1 仿 claude.ai 风格界面
- 完整深色模式（CSS 变量）
- Markdown 渲染（语法高亮、KaTeX 数学公式、Mermaid 图表）
- 图片附件缩略图和灯箱预览
- 文档预览面板（HTML / JSX 实时预览）
- Artifacts 系统与灵感画廊

### 6. 对话管理

**功能**：
- 创建、重命名、删除对话
- 对话历史持久化
- 支持多轮对话上下文
- 会话恢复（`--resume` 参数）

### 7. 设置与配置

**通用设置**：
- 主题切换（浅色/深色/自动）
- 聊天字体设置
- 默认模型选择
- 发送快捷键配置（Enter / Ctrl+Enter）

**账户设置**：
- 个人资料编辑
- 密码修改
- 会话管理
- 账户删除

**使用统计**：
- API 调用次数
- Token 使用量
- 会话统计

## 快速开始

### 方式一：下载安装包（推荐）

1. **下载安装包**

   访问 [Releases 页面](https://github.com/pretend1111/claude-desktop-app/releases/latest) 下载对应平台的安装包：

   | 平台 | 文件格式 |
   |------|---------|
   | Windows | `.exe` |
   | macOS | `.dmg` |
   | Linux | `.AppImage` / `.deb` |

2. **安装并启动**

   - Windows：双击 `.exe` 文件安装
   - macOS：打开 `.dmg` 文件，拖拽到应用程序文件夹
   - Linux：给 `.AppImage` 添加执行权限后运行，或安装 `.deb` 包

3. **配置 API**

   - 点击右上角齿轮图标进入设置
   - 选择 **Models** 标签
   - 点击 **Add Provider** 添加提供商
   - 输入 Base URL 和 API Key
   - 选择模型并保存

4. **开始使用**

   在输入框输入你的问题或任务，AI 会自动调用工具完成任务。

### 方式二：从源码构建

**前置要求**：
- Node.js 20+
- Bun

**步骤**：

```bash
# 1. 克隆仓库
git clone https://github.com/pretend1111/claude-desktop-app.git
cd claude-desktop-app

# 2. 安装依赖
npm install
cd engine && bun install && cd ..

# 3. 构建前端
npm run build

# 4. 启动应用
npm run electron:dev
```

**开发模式**：
```bash
# 前端开发服务器
npm run dev

# Electron 开发模式
npm run electron:dev
```

## 详细功能说明

### 对话功能

#### 创建新对话
1. 点击左侧边栏的 **Chats** 标签
2. 点击 **New Chat** 按钮
3. 开始输入消息

#### 使用技能
1. 在输入框中输入 `/` 查看可用技能
2. 选择技能或输入 `/skill-name`
3. 技能会以蓝色标签显示
4. AI 会读取技能指令并执行

#### 上传文件
1. 点击输入框左侧的附件图标
2. 选择要上传的文件（图片、文档等）
3. 文件会作为消息的一部分发送
4. AI 可以读取和分析文件内容

#### 查看工具调用
- 工具操作会实时显示为卡片
- 点击卡片可展开查看详细信息
- 工具类型包括：Read、Write、Edit、Bash、WebSearch 等

### 项目功能

#### 创建项目
1. 点击左侧边栏的 **Projects** 标签
2. 点击 **New Project** 按钮
3. 输入项目名称和描述
4. 上传参考文档

#### 管理项目文档
1. 进入项目详情页
2. 点击 **Add Documents** 上传文件
3. 文档会自动索引，AI 可按需访问
4. 支持代码、Markdown、PDF 等格式

#### 项目指令
- 在项目设置中添加自定义指令
- 指令会注入到该项目的所有对话中
- 用于定义项目规范、代码风格等

### Artifacts 功能

**功能描述**：AI 生成的可交互内容（HTML、React 组件等）会在右侧面板实时预览。

**支持类型**：
- HTML 页面
- React/JSX 组件
- SVG 图形
- 数据可视化

**使用方式**：
1. 要求 AI 创建可视化内容
2. 内容会自动在右侧面板渲染
3. 可以实时编辑和预览
4. 保存到 Artifacts 画廊

### 设置功能

#### 模型配置

**添加提供商**：
1. 进入 **Settings > Models**
2. 点击 **Add Provider**
3. 填写信息：
   - Provider Name：提供商名称
   - Base URL：API 端点
   - API Key：你的密钥
4. 点击 **Test Connection** 测试连接
5. 保存配置

**选择模型**：
1. 在提供商列表中选择已配置的提供商
2. 从模型列表中选择模型
3. 设置为默认模型（可选）

**示例配置（Qwen）**：
```
Provider Name: Qwen
Base URL: https://dashscope.aliyuncs.com/compatible-mode
API Key: sk-xxxxxxxxxxxxx
Model: qwen-max
```

#### 主题设置

**切换主题**：
1. 进入 **Settings > General**
2. 在 **Theme** 下拉菜单选择：
   - Light：浅色模式
   - Dark：深色模式
   - Auto：跟随系统

**自定义字体**：
- 在 **Chat Font** 选项中选择字体
- 支持系统默认字体和等宽字体

#### 快捷键配置

**发送消息快捷键**：
- Enter：直接发送
- Ctrl+Enter：发送消息

**换行快捷键**：
- Shift+Enter：插入换行
- Enter：插入换行（当发送键为 Ctrl+Enter 时）

## 配置指南

### 环境变量配置

应用支持通过 `.env` 文件配置引擎参数：

```bash
# API 配置
ANTHROPIC_MODEL=claude-opus-4-6
ANTHROPIC_DEFAULT_SONNET_MODEL=claude-sonnet-4-6
ANTHROPIC_DEFAULT_HAIKU_MODEL=claude-haiku-4-5
ANTHROPIC_DEFAULT_OPUS_MODEL=claude-opus-4-6

# 超时设置
API_TIMEOUT_MS=120000

# 功能开关
DISABLE_TELEMETRY=true
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=true
CLAUDE_CODE_DISABLE_THINKING=false
```

### 技能开发

**创建自定义技能**：

1. 使用内置的 `/skill-creator` 技能
2. 或手动创建技能文件夹：

```bash
mkdir -p ~/.claude/skills/my-skill
```

3. 创建 `SKILL.md` 文件：

```markdown
---
name: my-skill
description: 我的自定义技能
---

# 技能说明

这是一个自定义技能，用于...

## 使用方式

当用户输入...时，你应该...

## 示例

用户：请帮我...
助手：好的，我会...
```

4. 重启应用，技能会自动加载

### 项目配置

**项目结构**：
```
~/.claude/projects/
├── project-1/
│   ├── documents/
│   │   ├── doc1.md
│   │   └── doc2.pdf
│   └── project.json
└── project-2/
    └── ...
```

**project.json 示例**：
```json
{
  "name": "我的项目",
  "description": "项目描述",
  "instructions": "项目级指令，会注入到每个对话",
  "documents": [
    "documents/doc1.md",
    "documents/doc2.pdf"
  ]
}
```

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | React 19, TypeScript, TailwindCSS |
| 构建 | Vite 6 |
| 桌面 | Electron 41 |
| 引擎 | Claude Code (TypeScript, Bun) |
| 代理 | Node.js HTTP Server |
| Markdown | react-markdown, highlight.js, KaTeX, Mermaid |

### 数据流

1. **用户输入** → React 前端
2. **前端** → Bridge Server (`/api/chat`)
3. **Bridge Server** → 启动 Claude Code 引擎子进程
4. **引擎** → 调用 AI API（OpenAI 格式）
5. **Bridge Server** → 转换为 Anthropic 格式（如需要）
6. **AI 响应** → 引擎处理工具调用
7. **工具结果** → 流式返回前端
8. **前端** → 实时渲染工具卡片和响应

### OpenAI ↔ Anthropic 转换

**Bridge Server 自动处理**：
- 工具调用格式转换
- 思考模式映射（`enable_thinking` ↔ thinking blocks）
- 图像块注入
- 工具错误时的智能思考回退

## 常见问题

### 安装与启动

**Q: Windows 提示"无法验证发布者"？**

A: 这是正常的，因为应用没有代码签名证书。点击"更多信息" → "仍要运行"即可。

**Q: macOS 提示"无法打开，因为它来自身份不明的开发者"？**

A: 右键点击应用 → 选择"打开" → 点击"打开"确认。或在系统偏好设置中允许。

**Q: Linux 下 AppImage 无法运行？**

A: 添加执行权限：
```bash
chmod +x Claude-Desktop-*.AppImage
./Claude-Desktop-*.AppImage
```

### 配置问题

**Q: 如何配置 Qwen API？**

A: 
1. 进入 Settings > Models
2. Base URL: `https://dashscope.aliyuncs.com/compatible-mode`
3. API Key: 从阿里云控制台获取
4. Model: 选择 `qwen-max` 或其他模型

**Q: API 调用失败怎么办？**

A: 检查：
1. Base URL 是否正确
2. API Key 是否有效
3. 网络连接是否正常
4. 查看控制台错误日志

**Q: 如何使用本地模型（Ollama）？**

A:
1. 启动 Ollama 服务
2. Base URL: `http://localhost:11434/v1`
3. API Key: 随意填写（Ollama 不需要）
4. Model: 选择已下载的模型

### 使用问题

**Q: AI 无法执行命令？**

A: 确保：
1. 系统已安装必要的命令行工具
2. 应用有足够的权限
3. 工作目录正确

**Q: 技能无法加载？**

A: 检查：
1. 技能文件夹在 `~/.claude/skills/` 下
2. `SKILL.md` 文件格式正确
3. 重启应用

**Q: 项目文档无法访问？**

A: 确保：
1. 文件已正确上传
2. 文件格式支持（文本、代码、PDF）
3. 文件大小在限制内

### 性能问题

**Q: 响应速度慢？**

A: 
1. 检查网络连接
2. 尝试更换 API 提供商
3. 减少上下文长度
4. 使用更快的模型（如 Haiku）

**Q: 内存占用高？**

A: 
1. 关闭不需要的对话
2. 清理项目文档
3. 重启应用

### 开发问题

**Q: 如何调试？**

A: 
1. 打开开发者工具：Ctrl+Shift+I (Windows/Linux) 或 Cmd+Option+I (macOS)
2. 查看 Console 标签的错误信息
3. 查看 Network 标签的 API 请求

**Q: 如何贡献代码？**

A:
1. Fork 仓库
2. 创建功能分支
3. 提交 Pull Request
4. 遵循代码规范
