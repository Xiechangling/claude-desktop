- **布局**: 单栏布局，仅显示 chat 列表
- **设计**: 简洁的列表视图，搜索框，操作菜单

## 关键发现

### 1. 布局差异
- **当前**: 三栏布局（左 300px + 中 flex-1 + 右 400px）
- **官方**: 两栏布局（左侧边栏 + 主内容区）
- **问题**: 右侧 diff 查看器应该如何处理？

### 2. 左侧边栏差异
- **当前**: 宽边栏（300px），包含详细信息
- **官方浅色**: 宽边栏，包含 Projects 和 Sessions 分组
- **官方深色**: 窄边栏（~60-80px），仅图标
- **问题**: 应该使用哪种设计？

### 3. 输入框位置
- **当前**: 在主内容区底部，Context Selector 在输入框上方
- **官方浅色**: 输入框居中显示，下方有 "Select folder" 和 "Local" 按钮
- **官方深色**: 输入框在底部，Context Selector 在输入框上方
- **一致性**: 两个官方截图的输入框位置不同！

### 4. Context Selector
- **官方浅色**: 没有显示 Context Selector（可能在输入框下方的 "Select folder" 和 "Local" 中）
- **官方深色**: 在输入框上方，芯片式设计（Local, app, main, worktree）
- **问题**: 应该使用哪种设计？

### 5. Auto Accept
- **官方浅色**: 在输入框下方，左侧显示 "+ Auto accept edits"，右侧显示 "Sonnet 4.6"
- **官方深色**: 在输入框下方，左侧显示 "Auto accept edits"，右侧显示 "Opus 4.6"
- **一致性**: 位置一致，都在输入框下方

## 需要澄清的问题

1. **布局选择**: 应该使用两栏还是三栏布局？如果是两栏，右侧的 diff 查看器应该如何显示？
2. **左侧边栏**: 应该使用宽边栏（浅色主题）还是窄边栏（深色主题）？
3. **输入框位置**: 应该居中显示（浅色主题）还是在底部（深色主题）？
4. **Context Selector**: 应该在输入框上方（深色主题）还是在输入框下方作为按钮（浅色主题）？
5. **Projects 分组**: 浅色主题显示了 Projects 分组，当前实现没有。是否需要添加？

## 设计冲突

官方的两个截图（浅色和深色主题）显示了不同的设计：
- **浅色主题**: 宽边栏 + 居中输入框 + Projects 分组
- **深色主题**: 窄边栏 + 底部输入框 + Context Selector 芯片

这可能意味着：
1. 两个截图来自不同版本
2. 浅色和深色主题有不同的布局
3. 需要用户明确选择一个设计方向
